# Google Play Data Safety Form 回答ガイド

Google Play Console の Data Safety セクションで回答する内容をまとめたドキュメント。

## 概要

| 質問 | 回答 |
|------|------|
| アプリはユーザーデータを収集または共有しますか？ | はい |
| すべてのユーザーデータは転送中に暗号化されますか？ | はい（Firebase SDK は HTTPS を使用） |
| ユーザーがデータの削除をリクエストする方法はありますか？ | はい（アプリ内「設定」→「アカウント削除」） |

---

## 収集するデータの種類

### 1. デバイスまたはその他のID

| 項目 | 回答 |
|------|------|
| データの種類 | デバイスまたはその他のID |
| 詳細 | Firebase Anonymous UID、FCMトークン |
| 収集か共有か | 収集のみ（共有しない） |
| 一時的な処理か | いいえ（サーバーに保存） |
| 必須か任意か | 必須（アプリ機能に必要） |
| 目的 | アプリの機能、分析 |

### 2. アプリのアクティビティ

| 項目 | 回答 |
|------|------|
| データの種類 | アプリ内のユーザー生成コンテンツ |
| 詳細 | 習慣カード名、記録ログ、ニックネーム |
| 収集か共有か | 収集。一部共有（公開設定にした場合のみ） |
| 一時的な処理か | いいえ（サーバーに保存） |
| 必須か任意か | 必須（アプリのコア機能） |
| 目的 | アプリの機能 |

### 3. アプリの情報とパフォーマンス（Crashlytics 導入後）

| 項目 | 回答 |
|------|------|
| データの種類 | クラッシュログ |
| 詳細 | Firebase Crashlytics によるクラッシュレポート |
| 収集か共有か | 収集のみ |
| 一時的な処理か | いいえ |
| 必須か任意か | 必須 |
| 目的 | アプリの機能（安定性向上） |

---

## 収集しないデータ

以下のデータは **収集しません**：

- 位置情報
- 個人情報（名前、メールアドレス、電話番号、住所）
- 財務情報
- 健康・フィットネスデータ
- メッセージ（エールは定型文のみ）
- 写真・動画
- 音声
- ファイル
- カレンダー
- 連絡先
- ウェブ閲覧履歴

## データ共有について

| 質問 | 回答 |
|------|------|
| 第三者とデータを共有しますか？ | いいえ（Firebase はデータ処理者として利用） |
| 広告目的のデータ共有は？ | なし |

## セキュリティ

| 質問 | 回答 |
|------|------|
| データは転送中に暗号化されますか？ | はい |
| データの削除をリクエストできますか？ | はい（アプリ内から即時削除可能） |

## 補足

- Firebase SDK: Google LLC のデータ処理契約に基づき利用
- 匿名認証: メールやパスワード等の個人情報は不要
